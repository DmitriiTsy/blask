# 📚 Knowledge Base Storage

## Локальное хранение

Да, база знаний хранится **локально** на вашем компьютере в папке проекта.

### Структура хранения

```
Blask/
└── knowledge_base/              # Главная папка базы знаний
    ├── chroma.sqlite3          # Векторная база данных (ChromaDB)
    ├── documents/              # Папка с загруженными документами
    │   └── document1.pdf       # Ваши загруженные файлы
    └── documents_metadata.json # Метаданные документов (названия, даты, размеры)
```

### Что хранится:

1. **chroma.sqlite3** - векторная база данных
   - Содержит эмбеддинги (векторные представления) всех чанков документов
   - Используется для быстрого semantic search
   - Автоматически обновляется при добавлении новых документов

2. **documents/** - оригинальные файлы
   - Сохраняются копии всех загруженных документов
   - Форматы: PDF, TXT, MD
   - Используются для отображения и ссылок

3. **documents_metadata.json** - метаданные
   - Информация о каждом документе:
     - Имя файла
     - Дата загрузки
     - Размер файла
     - Количество чанков
     - Хеш файла (для дедупликации)

### Преимущества локального хранения:

✅ **Приватность** - все данные остаются на вашем компьютере  
✅ **Скорость** - быстрый доступ без интернета  
✅ **Контроль** - вы полностью контролируете свои данные  
✅ **Переносимость** - можно скопировать папку `knowledge_base/` на другой компьютер  

### Недостатки:

⚠️ **Только локально** - не синхронизируется между устройствами  
⚠️ **Нужно бэкапить** - если удалите папку, данные потеряются  
⚠️ **Размер** - может занимать место на диске (особенно с большими документами)  

### Управление базой знаний:

#### Просмотр содержимого:
```bash
ls -lh knowledge_base/
ls -lh knowledge_base/documents/
cat knowledge_base/documents_metadata.json
```

#### Резервное копирование:
```bash
# Создать бэкап
tar -czf knowledge_base_backup.tar.gz knowledge_base/

# Восстановить из бэкапа
tar -xzf knowledge_base_backup.tar.gz
```

#### Очистка базы знаний:
```bash
# Удалить все документы (осторожно!)
rm -rf knowledge_base/
```

### Размер базы знаний:

- **Векторная БД (chroma.sqlite3)**: ~несколько МБ на документ
- **Документы**: оригинальный размер файлов
- **Метаданные**: несколько КБ

### Перенос на другой компьютер:

1. Скопируйте папку `knowledge_base/` целиком
2. Вставьте в новый проект в корневую директорию
3. База знаний будет работать автоматически

### Важно:

- Папка `knowledge_base/` должна быть в `.gitignore` (если не хотите коммитить документы)
- При деплое на сервер база знаний будет создана автоматически
- Для продакшена можно настроить внешнее хранилище (S3, Google Cloud Storage и т.д.)
