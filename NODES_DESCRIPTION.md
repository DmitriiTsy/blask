# Описание нод проекта Blask

## 🏗️ Архитектура графа

```
START → Thinking Node → [Search Node] → Analysis Node → END
                          (опционально)
```

---

## 📋 Ноды проекта

### 1. 🤔 Thinking Node (Думающая нода)

**Расположение:** `src/nodes/thinking_node.py`

**Что делает:**
- ✅ Анализирует пользовательский запрос через LLM
- ✅ Принимает решение о типе запроса:
  - `"search"` - нужен поиск информации
  - `"direct_answer"` - можно ответить напрямую
  - `"statistics"` - нужна статистика и графики
- ✅ Определяет тип поиска:
  - `"keywords"` - поиск по ключевым словам
  - `"competitors"` - анализ конкурентов
  - `"trends"` - анализ трендов
- ✅ Формирует оптимизированный поисковый запрос
- ✅ Определяет необходимость графиков

**Входные данные:**
- `user_query` - запрос пользователя
- `conversation_history` - история разговора (опционально)

**Выходные данные:**
- `decision` - решение о типе запроса
- `search_type` - тип поиска (если нужен)
- `search_query` - оптимизированный запрос
- `needs_charts` - нужны ли графики
- `reasoning` - обоснование решения

**Маршрутизация:**
- Если `decision == "direct_answer"` → идет в Analysis Node (пропускает поиск)
- Если `decision == "search"` или `"statistics"` → идет в Search Node

---

### 2. 🔍 Search Node (Нода поиска)

**Расположение:** `src/nodes/search_node.py`

**Что делает:**
- ✅ Выполняет поиск в зависимости от типа:
  - **Keywords** - обычный поиск по ключевым словам через SerpAPI
  - **Competitors** - анализ конкурентов через CompetitorAnalyzer
  - **Trends** - анализ трендов через TrendAnalyzer
- ✅ Собирает данные из различных источников
- ✅ Обрабатывает результаты поиска

**Входные данные:**
- `search_type` - тип поиска (keywords/competitors/trends)
- `search_query` - запрос для поиска

**Выходные данные:**
- `search_results` - список результатов поиска
- `raw_data` - сырые данные для дальнейшей обработки

**Текущая реализация:**
- Использует `SearchBasedTrendAnalyzer` - делает обычный поиск вместо специального Google Trends API
- Fallback на mock поиск, если API ключи не настроены

**Маршрутизация:**
- Всегда идет в Analysis Node после выполнения поиска

---

### 3. 📊 Analysis Node (Нода анализа)

**Расположение:** `src/nodes/analysis_node.py`

**Что делает:**
- ✅ Обрабатывает результаты поиска
- ✅ Создает визуализации (графики) если нужно
- ✅ Форматирует ответ для пользователя через LLM
- ✅ Структурирует данные в удобный формат

**Входные данные:**
- `search_results` - результаты поиска
- `raw_data` - сырые данные
- `needs_charts` - нужны ли графики

**Выходные данные:**
- `formatted_response` - отформатированный ответ
- `visualization` - график (base64 encoded image)
- `charts_created` - флаг создания графиков
- `processed_data` - обработанные данные

**Маршрутизация:**
- Финальная нода → END

## 🎯 Фокус на трендах и лучших решениях

### Текущие возможности для трендов:

1. **Thinking Node** определяет `search_type="trends"`
2. **Search Node** использует `TrendAnalyzer` для получения трендов
3. **Analysis Node** форматирует результаты

### Ограничения:

- ❌ Используется обычный поиск вместо специального Google Trends API
- ❌ Нет анализа временных рядов (interest over time)
- ❌ Нет связанных запросов (related queries)
- ❌ Нет регионального анализа
- ❌ Нет сравнения трендов
