# Blask - LangGraph приложение для анализа трендов и конкурентов

Система на базе LangGraph для анализа трендов, конкурентов и статистики с использованием думающих нод, поиска и визуализации данных.

## 🚀 Быстрый старт

### 1. Установка зависимостей

**Вариант 1: Автоматическая установка (рекомендуется)**
```bash
./install.sh
```

**Вариант 2: Ручная установка**
```bash
# Создать виртуальное окружение
python3 -m venv venv

# Активировать окружение
source venv/bin/activate  # macOS/Linux
# или
venv\Scripts\activate     # Windows

# Установить зависимости
pip install -r requirements.txt
```

**Если pytest не найден:**
```bash
# Используйте python3 -m pytest вместо pytest
python3 -m pytest tests/
```

Подробнее см. [INSTALL.md](./INSTALL.md)

### 2. Настройка окружения

**Вариант 1: Автоматическое создание .env**
```bash
./SETUP_ENV.sh
# Затем отредактируйте .env и добавьте ваш API ключ
nano .env
```

**Вариант 2: Ручное создание**
```bash
# Создать .env файл
touch .env

# Отредактировать
nano .env
```

**Минимальная конфигурация (.env файл):**
```env
OPENAI_API_KEY=sk-your-key-here
```

**Все переменные:**
- `OPENAI_API_KEY` или `ANTHROPIC_API_KEY` - **ОБЯЗАТЕЛЬНО** (минимум один)
- `SERPAPI_KEY` - опционально (для реального поиска)
- `GOOGLE_API_KEY` и `GOOGLE_CSE_ID` - опционально

📖 Подробнее: [ENV_VARIABLES.md](./ENV_VARIABLES.md)

### 3. Запуск проекта

#### 🌐 Веб-интерфейс (рекомендуется)

```bash
./run_webapp.sh
```

Или напрямую:
```bash
streamlit run src/webapp.py
```

Откроется веб-интерфейс по адресу: **http://localhost:8501**

#### 💻 Командная строка

**Вариант 1: С вашим запросом**
```bash
python3 -m src.main "Ваш запрос здесь"
```

**Вариант 2: Интерактивный режим**
```bash
python3 -m src.main
# Затем введите запрос когда появится приглашение
```

**Вариант 3: Через скрипт**
```bash
./run.sh "Ваш запрос"
```

**Примеры:**
```bash
# Поиск трендов
python3 -m src.main "Какие тренды в AI сейчас?"

# Анализ конкурентов
python3 -m src.main "Кто конкуренты в e-commerce?"

# Статистика с графиками
python3 -m src.main "Покажи статистику Python за год"
```

## 🏗️ Архитектура

Система состоит из трех основных нод:

1. **Thinking Node** - анализирует запрос и принимает решение
2. **Search Node** - выполняет поиск по ключевым словам и конкурентам
3. **Analysis Node** - обрабатывает данные и создает визуализации

Подробнее см. [PLAN.md](./PLAN.md)

## 🔧 Разработка

### Запуск тестов

```bash

source venv/bin/activate

# Запуск всех тестов с покрытием

pytest tests/ --cov=src --cov-fail-under=100
```

### Проверка типов

```bash
# Проверка типизации с mypy
mypy src/
```

### Структура кода

Проект следует принципам SOLID:
- **Single Responsibility** - каждый модуль отвечает за одну задачу
- **Open/Closed** - открыт для расширения, закрыт для модификации
- **Liskov Substitution** - подклассы заменяют базовые классы
- **Interface Segregation** - маленькие, специфичные интерфейсы
- **Dependency Inversion** - зависимость от абстракций, не от конкретных реализаций

Каждая нода - независимый модуль с четко определенными входными и выходными данными.

### Покрытие тестами

Проект требует **100% покрытия тестами**. Все компоненты должны иметь:
- Unit тесты для каждого класса/функции
- Тесты граничных случаев
- Тесты обработки ошибок
- Интеграционные тесты для графа
